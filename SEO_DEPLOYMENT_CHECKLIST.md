# SportsBettingPrime SEO Deployment Checklist

Permanent standard (June 3, 2026 indexing repair). Every publish/build/deploy
that touches pages, the sitemap, robots.txt, or generators MUST satisfy all of
this before being called complete.

## Sitemap policy (curated, not exhaustive)
- `sitemap.xml` is generated ONLY by `generate_sitemap.py` (root). Do not hand-edit.
- It contains ONLY: homepage, hubs, evergreen guides, dated feature articles,
  blog posts, core/policy pages. Target ~100-150 URLs.
- NEVER in the sitemap: `archive/**`, `consensus_library/**`,
  `handicapping-hub-archive/**`, `covers-consensus-YYYY-MM-DD.html` dailies,
  `sportsbettingprime-covers-consensus-*`, dated `handicapping-hub-*` backups,
  `404.html`, `google*.html`, `sitemap.html`, anything noindex/meta-refresh/
  non-self-canonical.
- Every sitemap URL must be: HTTP 200, zero redirects, no noindex (meta or
  X-Robots-Tag), self-referencing canonical, crawlable per robots.txt.

## Canonical rules
- Indexable pages: exactly one self-referencing canonical
  (`https://sportsbettingprime.com/<file>.html`, homepage = `/`). Never `www.`
- Daily covers-consensus archives: canonical -> `covers-consensus.html` hub.
  (`github_consensus_update.py` already emits this; `scripts/fix_seo_indexing_20260603.py`
  backfilled all historical dailies.)
- `archive/**` daily snapshots: canonical -> their live hub page (build_site.py
  archives the hub HTML, which carries the hub canonical - correct, keep).
- Near-duplicate pages: pick ONE canonical version; point the other at it
  (example: `0303-nba-playoff-race...` -> `nba-playoff-race-march-2026-standings-analysis.html`).

## Noindex rules
- `404.html` and `handicapping-hub-calendar.html` stay `noindex, follow`.
- Never noindex hubs, guides, dated feature articles, blog posts, or core pages.

## Titles / meta
- NO DATES in `<title>` (network rule). Dates live in body/calendars/FORCED dates.
- Unique title + meta description + H1 per indexable page. No two sitemap pages
  may share a title.
- Sport hub titles come from `scripts/build_site.py` `dateless_titles` map - keep dateless.

## handicapping-hub.html (special case)
- It is copied verbatim from the BetLegend pipeline. The copy step MUST be
  followed by `python scripts\fix_handicapping_hub_head.py` (wired into
  `DAILY_UPDATE_HANDICAPPING_HUB.bat` and `HANDICAPPING_HUB_AUTO.bat`).
  This sets the SBP canonical, robots, dateless SBP-branded title, and JSON-LD
  publisher. Never commit the hub without it.

## robots.txt
- `Allow: /` for all public content; sitemap line must stay:
  `Sitemap: https://sportsbettingprime.com/sitemap.xml`
- Never add Disallow rules covering articles, picks, guides, hubs, blog, archive
  calendars, or landing pages.

## Internal linking
- A new page is NOT complete until it has at least one inbound link from the
  correct live structure (homepage/latest/archive/hub/calendar) - network rule 17/18.
- Homepage must keep linking all sport hubs, covers-consensus, handicapping-hub,
  betting-guide, about, faq.

## Required verification before calling any deploy complete
```
python generate_sitemap.py
python scripts/verify_sitemap_indexability.py          # local gate (CI runs this too)
python scripts/seo_audit_sbp.py                        # full repo audit
# after push + GitHub Pages deploy:
python scripts/verify_sitemap_indexability.py --live   # live 200/redirect/noindex/canonical gate
```
All must exit 0 / report `failures=0`. The GitHub Action `daily-sports-update.yml`
runs the same gate; if it exits non-zero the sitemap won't ship - fix the page,
never the gate.

## Regression warnings (learned the hard way)
- The CI sitemap step was silently failing for weeks (2 missing canonicals),
  which was the only thing stopping a 1,097-URL bloated sitemap from shipping.
  `generate_sitemap.py` policy is now curated - do NOT revert it to
  "every content page" mode.
- `build_site.py` rebuilds the sport hubs' `<main>` and `<title>` daily. Evergreen
  SEO content on those pages must live OUTSIDE `<main>` or it will be clobbered.
- `archive-calendar.html` is regenerated by `build_site.py` - its canonical/meta
  live in the TEMPLATE inside build_site.py, not the HTML file.
